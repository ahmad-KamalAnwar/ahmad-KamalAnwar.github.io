Flagship

Help Desk — WorkHub

Role Architect & Lead Backend Engineer · built end to end

Stack Laravel · OpenSearch · MySQL · React · Multi-tenant · AWS

What it is

Customer support hub for the WorkHub suite — ticketing, email integration, and automated workflows for businesses managing support at scale.

What I built

I designed this from a blank schema. That meant the data model, the ticketing engine, the email ingestion pipeline, the workflow automation rules, and the full API surface — then the frontend views on top of it. Nothing was inherited from a legacy system; every architectural decision was mine to make.

The hard part

SQL full-text search was falling apart under real ticket volumes — too slow for agents actually doing their jobs. I replaced it with OpenSearch and got search down to sub-second response times. Also designed the schema for multi-tenancy from day one, which saved a costly retrofitting exercise later when the customer base grew.

Outcome

Launched and directly lifted sales conversion and activation at onboarding. Probably the most complete end-to-end delivery I've done — I owned it from the first whiteboard sketch to production.

GeoPlugin

Role Senior Backend Engineer · Devsloop · production API ownership

Stack PHP · Laravel · REST API · Geolocation · AWS · PostgreSQL

What it is

Public IP geolocation and reverse geocoding API — developers use it for personalisation, compliance, geo-targeting, and anything that needs to know where a request is coming from.

What I built

Led the architecture and core feature delivery at Devsloop. The product lives or dies by reliability and latency — integrators plug this into production code and expect it to just work. So that's what I optimised for: a clean, predictable API contract, a database and caching layer tuned for read-heavy low-latency workloads, and infrastructure that doesn't surprise you at 2am.

What mattered

The success metric here wasn't feature count — it was p95 latency and uptime. I reported against both.

Moderation, Tagging & Data Lake — eKomi

Role Software Engineer → Senior · Coeus Solutions GmbH · Berlin · 4 years

Stack Symfony · PHP · MySQL · Microservices · REST API · Reporting

What it is

eKomi is an independent customer feedback and reviews platform. The Moderation, Tagging, and Data Lake systems sit at the core of how their clients collect, process, and analyse customer feedback at enterprise scale.

Clients on the platform

These systems handled feedback data for brands across insurance, travel, retail, automotive, and finance — including AXA, SIXT, Bookatable by Michelin, GetYourGuide, compare.com, ERGO, Esprit, Douglas, TGW Travel Group, Euro Taxis, and others.

What I built

I joined as an associate and spent four years across these three products. For Moderation I added graph-based reporting. For Tagging I stabilised an architecture with real structural debt — decoupled the processing pipeline from the API surface so both could evolve without breaking live integrations. For Data Lake I revamped the reporting layer to surface cross-service data into dashboards that made sense to the people reading them.

The hard part

Refactoring a system with active enterprise clients on it — you can't break what AXA or SIXT is relying on while you're fixing the internals. Required careful incremental work: decouple, test, verify, release. No big-bang rewrites.

Outcome

Grew from associate to senior over four years on the same client. Longest and most architecturally complex engagement I've had.
